Complete guide to integrate Firebase with KrishiSetu Admin Dashboard

1

Create Firebase Project

  1. Go to Firebase Console
  2. Click "Create a project" or "Add project"
  3. Enter project name: krishisetu-admin
  4. Choose whether to enable Google Analytics (optional)
  5. Click "Create project"
‚úÖ Success: Your Firebase project is now created!
2

Enable Firestore Database

  1. In your Firebase project console, click "Firestore Database" in the left sidebar
  2. Click "Create database"
  3. Choose "Start in test mode" (for development)
  4. Select a location for your database (choose closest to your users)
  5. Click "Done"
‚ö†Ô∏è Security Rules: Test mode allows read/write access to all users. For production, you'll need to configure proper security rules.
3

Get Firebase Configuration

  1. In your Firebase project console, click the gear icon ‚öôÔ∏è and select "Project settings"
  2. Scroll down to "Your apps" section
  3. Click the web icon "</>" to add a web app
  4. Enter app nickname: KrishiSetu Admin
  5. Click "Register app"
  6. Copy the Firebase configuration object

Your Firebase Config will look like this:

const firebaseConfig = { apiKey: "AIzaSyC...", authDomain: "krishisetu-admin.firebaseapp.com", projectId: "krishisetu-admin", storageBucket: "krishisetu-admin.appspot.com", messagingSenderId: "123456789", appId: "1:123456789:web:abcdef123456" };
4

Update Firebase Configuration in HTML

Replace the placeholder Firebase config in admin-dashboard-firebase.html:

// Replace this section in admin-dashboard-firebase.html const firebaseConfig = { apiKey: "your-api-key-here", // ‚Üê Replace with your actual API key authDomain: "your-project-id.firebaseapp.com", // ‚Üê Replace with your domain projectId: "your-project-id", // ‚Üê Replace with your project ID storageBucket: "your-project-id.appspot.com", // ‚Üê Replace with your storage bucket messagingSenderId: "123456789", // ‚Üê Replace with your sender ID appId: "your-app-id" // ‚Üê Replace with your app ID };
‚ö†Ô∏è Important: Make sure to replace ALL placeholder values with your actual Firebase configuration values.

Configure Security Rules (Production)

For production, update your Firestore security rules:

  1. Go to Firestore Database in Firebase Console
  2. Click on "Rules" tab
  3. Replace the default rules with:
rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // Allow read/write access to admin users only match /{document=**} { allow read, write: if request.auth != null && request.auth.token.admin == true; } // Allow farmers to read their own data match /farmers/{farmerId} { allow read: if request.auth != null && request.auth.uid == farmerId; } // Allow companies to read their own data match /companies/{companyId} { allow read: if request.auth != null && request.auth.uid == companyId; } } }
‚ö†Ô∏è Note: These rules require Firebase Authentication. For now, keep test mode enabled during development.

  4. Try adding a new farmer or company
  5. Check your Firebase Console to see if data appears in Firestore
‚úÖ Success Indicators:
  • Firebase connection status shows "Connected"
  • Data appears in Firebase Console ‚Üí Firestore Database
  • Real-time updates work (data syncs automatically)
  • No console errors in browser developer tools
